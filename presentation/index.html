Phase 1 — Requirements → Wireframes

Turn Any Requirement
Into a Figma-Ready Wireframe

Feed the plugin a Jira ticket, PDF, Word doc, or plain text. It reads the requirements, makes autonomous UX decisions, and outputs pixel-perfect greyscale wireframes ready to import into Figma — plus a FigJam user-flow diagram.

What Claude decides for you
  • →
    Navigation pattern (top nav for ≤4 screens, sidebar for 5–9+)
  • →
    Color palette by domain (SaaS, healthcare, finance…)
  • →
    Typography, spacing, density, border-radius
# Point at your requirements source /requirements-to-figma jira:PROJ-42 # Or a file /requirements-to-figma ./specs/prd.pdf # Output artifacts: wireframes/*.html # Greyscale mockups .claude/wireframe-spec.json .claude/design-tokens.css # + FigJam user-flow diagram auto-created
Jira Tickets PDFs Word Docs Plain Text FigJam Diagrams
Phase 2 — Push to Figma

Send Wireframes to Figma
Directly from the CLI

One command pushes every generated HTML wireframe into a Figma file as editable frames — no manual html.to.design import, no drag-and-drop. Designers open Figma and the wireframes are already there.

One-time terminal setup (run once per machine)
# A. Install Figma plugin & log in claude plugin install figma@claude-plugins-official # B. Connect Figma remote MCP claude mcp add --transport http figma-remote-mcp \ https://mcp.figma.com/mcp
# Push a single wireframe to a specific Figma node /push-to-figma \ --figma "figma.com/design/TFKVd.../Demo?node-id=134-2" \ --html ".claude/wireframes/html/03-dashboard.html" # Push all wireframes to a Figma file /push-to-figma \ --figma "figma.com/design/TFKVd.../Demo" \ --html ".claude/wireframes/html/" # Push FigJam user-flow diagram only /push-to-figma \ --figma "figma.com/design/TFKVd.../Demo" \ --figjam-only
Single file or folder node-id targeting FigJam diagrams Auto frame naming

After /push-to-figma, the wireframes are already on the Figma canvas. Designers apply visual polish on top — no re-importing, no re-drawing from scratch.

Pro tip for designers: Use Auto Layout, name your layers clearly, and use Named Styles — the plugin reads all of these and maps them directly to CSS.
Phase 4 — Setup

Configure Once,
Generate Forever

The /setup-wizard command detects your Angular project, asks a few questions, and configures everything — so every subsequent command just works.

  • ✓
    Angular version — 14, 15, 16, or 17 (syntax adapts automatically)
  • ✓
    Output directory — where components land
  • ✓
    Stylesheet format — SCSS, CSS, or Less
  • ✓
    Reference project — inherit naming conventions from existing codebase
  • ✓
    Unit tests — generate spec files or skip
# Run once per project /setup-wizard # Or install into new project bash scripts/install.sh --new my-app # Or add to existing project bash scripts/install.sh ./my-app # Config saved to: .claude/figma-generator.config.json
Angular 14–17 Windows / macOS / Linux SCSS / CSS / Less
Phase 5 — Backend API

Generate Type-Safe Services
from Any API Contract

Point the plugin at your backend — an OpenAPI spec, a NestJS project, a Spring Boot controller, or even a plain description — and it generates a complete Angular integration layer.

  • ✓
    TypeScript interfaces — every DTO / model typed precisely
  • ✓
    HTTP services — one per domain, typed Observable<T>
  • ✓
    JWT Auth interceptor — auto-attaches Bearer tokens
  • ✓
    Error interceptor — 401→logout, 403→forbidden, 0→network error
  • ✓
    Dev proxy config — CORS-free local development
  • ✓
    CORS handoff doc — instructions for your backend team
/api-to-components ./api/openapi.yaml # Also works with: /api-to-components ./backend/src/ # NestJS /api-to-components ./java-api/src/ # Spring Boot /api-to-components ./fastapi/main.py # FastAPI # Generated output: src/app/core/services/ src/app/core/models/ src/app/core/interceptors/ src/environments/ proxy.conf.json .claude/cors-handoff.md
Phase 6 — Code Generation

Paste a Figma URL.
Get Angular Components.

Three generation modes to fit every use case. The plugin calls Figma's MCP API, reads the full design spec, and generates production-ready Angular code — no token setup required.

# One reusable component /generate-component figma.com/design/.../node-id=42:1 # Full page + all children + route /generate-page figma.com/design/.../node-id=10:5 # Every frame in the file at once /generate-all figma.com/design/AbCdEf12345/app
  • ✓
    Auto Layout → CSS Flexbox/Grid — exact spacing preserved
  • ✓
    Colors & typography — Figma Named Styles → CSS custom properties
  • ✓
    Angular 17 signals — input<T>(), @if, @for syntax
  • ✓
    Standalone components — always, no NgModule boilerplate
  • ✓
    BEM class naming — .card__title--active
  • ✓
    Accessibility — aria-label, alt text, semantic HTML
  • ✓
    Optional unit tests — .spec.ts with TestBed scaffold
Phase 7 — Assets

Images & Icons
Downloaded Automatically

The moment code is generated, a PostToolUse hook fires. It scans the Figma design for all images and icons, downloads them, optimizes them, and places them at the correct Angular asset paths — before you even look at the generated file.

🔍
Detect
Scan Figma nodes for images (RECTANGLE) and icons (VECTOR ≤64px)
⬇️
Download
Fetch PNG/SVG from Figma using the export API with auth token
✂️
Optimize
Run SVG through svgo and PNG through optipng — smaller bundles
📁
Organize
src/assets/images/ and src/assets/icons/ — correct Angular paths
Generated Angular code always references /assets/… paths — never Figma CDN URLs that expire.
