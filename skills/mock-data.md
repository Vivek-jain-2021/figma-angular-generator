# Skill: Mock Data Generation

Reference guide for generating realistic, fully-typed mock data and HTTP interceptor layers for Angular projects.

---

## File Layout

```
src/app/core/
  mock/
    mock-data.ts              ← all mock arrays and response objects
    mock-api.interceptor.ts   ← HTTP interceptor that serves mock data
src/environments/
  environment.ts              ← useMockData: true  (dev)
  environment.prod.ts         ← useMockData: false (prod)
```

---

## Environment Toggle Pattern

```typescript
// src/environments/environment.ts
export const environment = {
  production: false,
  apiUrl: '/api',
  useMockData: true,    // flip to false to hit the real backend
  mockDelay: 300        // simulated network latency in ms
};

// src/environments/environment.prod.ts
export const environment = {
  production: true,
  apiUrl: 'https://api.yourapp.com',
  useMockData: false,   // always false in production
  mockDelay: 0
};
```

## Realistic Field Generation Rules

Use these rules when populating mock records to produce plausible, readable data.

### IDs

```typescript
// Prefix-based sequential string IDs (preferred for readability)
id: 'ord-001', 'ord-002', ...
id: 'prd-001', 'prd-002', ...
id: 'usr-001', 'usr-002', ...

// UUID pattern (when model uses UUID format)
id: 'a1b2c3d4-e5f6-7890-abcd-ef1234567890'
```

### Dates (ISO 8601)

```typescript
// Spread across last 90 days — most recent first
createdAt: '2025-03-10T14:32:00Z',
updatedAt: '2025-03-11T09:15:00Z',

// Use realistic time-of-day spread (not all midnight)
// Business hours range: 08:00–18:00 UTC
```

### Enums / Union Types

```typescript
// Cycle every value across the dataset so all states are testable
// e.g. status: 'pending' | 'processing' | 'shipped' | 'delivered' | 'cancelled'
// With 10 records:
// records 0,5 → 'pending'; 1,6 → 'processing'; 2,7 → 'shipped'; 3,8 → 'delivered'; 4,9 → 'cancelled'

function cycleEnum<T>(values: T[], index: number): T {
  return values[index % values.length];
}
```

### Numbers

| Field name contains | Realistic range |
|---------------------|----------------|
| `price`, `amount`, `total`, `cost` | 5.00 – 500.00 (2 decimal places) |
| `quantity`, `count`, `qty` | 1 – 20 |
| `rating`, `score` (out of 5) | 1 – 5 |
| `rating`, `score` (out of 100) | 60 – 100 |
| `percentage`, `percent`, `rate` | 0 – 100 |
| `age` | 18 – 65 |
| `page` | 1 |
| `size`, `limit` | 10 or 20 |
| `total` in list responses | match array length |
| `lat`, `lng`, `latitude`, `longitude` | realistic geo coords |

### Strings

| Field name contains | Realistic content |
|--------------------|-------------------|
| `name` (person) | "Alice Hartley", "Carlos Mendez", "Priya Nair", "James Okonkwo", "Sofia Larsson" |
| `name` (product) | "Wireless Keyboard", "Standing Desk", "Noise Cancelling Headphones", "USB-C Hub", "Ergonomic Chair" |
| `title` | Short, capitalized descriptive titles |
| `description` | 1–2 sentences, domain-appropriate |
| `email` | firstname.lastname@example.com |
| `phone` | "+1-555-0100" through "+1-555-0109" |
| `address` | "123 Main St, Springfield, IL 62701" |
| `url`, `imageUrl`, `avatarUrl` | Use `/assets/images/placeholder.jpg` (never external URLs) |
| `slug` | kebab-case version of title/name |
| `token`, `apiKey` | "mock-token-0001" (never real keys) |
| `message`, `note`, `comment` | Short realistic sentence |
| `currency` | "USD" |
| `locale` | "en-US" |

### Booleans

```typescript
// Alternate true/false across records so both states are testable
isActive: index % 2 === 0,
isVerified: index % 3 !== 0,
```

### Nested Objects

Always populate nested objects fully. Never leave a required nested field as `null` unless the interface marks it optional.

```typescript
// ✓ Correct — fully populated
items: [
  { productId: 'prd-001', name: 'Wireless Keyboard', quantity: 2, unitPrice: 79.99 }
],

// ✗ Wrong — empty or null nested objects hide bugs
items: [],
address: null,
```

## Paginated List Response Pattern

When the service returns a `*ListResponse` type, mock the pagination properly:

```typescript

if (url.includes('/orders') && method === 'GET') {
  const page  = +(req.params.get('page')  ?? 1);
  const size  = +(req.params.get('size')  ?? 20);

  const data  = MOCK_ORDERS.slice(start, start + size);
  const body  = { data, total: MOCK_ORDERS.length, page, size };
  return of(new HttpResponse({ status: 200, body })).pipe(delay(MOCK_DELAY));
```

## Functional Interceptor Pattern (Angular 15–17)

```typescript
// src/app/core/mock/mock-api.interceptor.ts
import { HttpInterceptorFn, HttpResponse } from '@angular/common/http';
import { of } from 'rxjs';
import { delay } from 'rxjs/operators';
import { environment } from '../../../environments/environment';
import { MOCK_ORDERS, MOCK_ORDERS_LIST, MOCK_AUTH_RESPONSE, MOCK_USERS } from './mock-data';

const MOCK_DELAY = environment.mockDelay ?? 300;

export const mockApiInterceptor: HttpInterceptorFn = (req, next) => {
  // Guard — pass through when mock mode is off
  if (!environment.useMockData) {
    return next(req);
  }

  const url    = req.url;
  const method = req.method;

  // Helper to return a mock response
  const respond = <T>(body: T, status = 200) =>
    of(new HttpResponse<T>({ status, body })).pipe(delay(MOCK_DELAY));

  // ── Auth ──────────────────────────────────────────────────────────────────
  if (url.includes('/auth/login')  && method === 'POST') return respond(MOCK_AUTH_RESPONSE);
  if (url.includes('/auth/logout') && method === 'POST') return respond(null, 204);
  if (url.includes('/auth/me')     && method === 'GET')  return respond(MOCK_USERS[0]);

  // ── Orders ────────────────────────────────────────────────────────────────
  if (url.match(/\/orders\/[\w-]+$/) && method === 'GET') {
    const id   = url.split('/').pop()!;
    const item = MOCK_ORDERS.find(o => o.id === id) ?? MOCK_ORDERS[0];
    return respond(item);
  }
  if (url.includes('/orders') && method === 'GET') {
    const page  = +(req.params.get('page')  ?? 1);
    const size  = +(req.params.get('size')  ?? 20);
    const start = (page - 1) * size;
    const data  = MOCK_ORDERS.slice(start, start + size);
    return respond({ data, total: MOCK_ORDERS.length, page, size });
  }
  if (url.includes('/orders') && method === 'POST') {
    const newItem = { ...MOCK_ORDERS[0], ...req.body as object, id: `ord-${Date.now()}`, createdAt: new Date().toISOString() };
    return respond(newItem, 201);
  }
  if (url.match(/\/orders\/[\w-]+$/) && (method === 'PUT' || method === 'PATCH')) {
    const id      = url.split('/').pop()!;
    const current = MOCK_ORDERS.find(o => o.id === id) ?? MOCK_ORDERS[0];
    return respond({ ...current, ...req.body as object, updatedAt: new Date().toISOString() });
  }
  if (url.match(/\/orders\/[\w-]+$/) && method === 'DELETE') {
    return respond(null, 204);
  }

  // ── Fallthrough — pass to real backend ────────────────────────────────────
  return next(req);
};
```

## Class-Based Interceptor Pattern (Angular 14)

```typescript
// src/app/core/mock/mock-api.interceptor.ts
import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent, HttpResponse } from '@angular/common/http';
import { Observable, of } from 'rxjs';
import { delay } from 'rxjs/operators';
import { environment } from '../../../environments/environment';
import { MOCK_ORDERS, MOCK_ORDERS_LIST } from './mock-data';

@Injectable()
export class MockApiInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<unknown>, next: HttpHandler): Observable<HttpEvent<unknown>> {
    if (!environment.useMockData) {
      return next.handle(req);
    }

    const url    = req.url;
    const method = req.method;
    const ms     = environment.mockDelay ?? 300;

    if (url.includes('/orders') && method === 'GET') {
      return of(new HttpResponse({ status: 200, body: MOCK_ORDERS_LIST })).pipe(delay(ms));
    }

    // Add more matchers for each domain ...

    return next.handle(req);
  }
}
```

## Interceptor Registration

### Angular 15–17 (`app.config.ts`)

```typescript
import { provideHttpClient, withInterceptors } from '@angular/common/http';
import { mockApiInterceptor } from './core/mock/mock-api.interceptor';
import { authInterceptor }    from './core/interceptors/auth.interceptor';
import { errorInterceptor }   from './core/interceptors/error.interceptor';
import { environment }        from '../environments/environment';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient(
      withInterceptors([
        // Mock MUST be first — it short-circuits before auth runs
        ...(environment.useMockData ? [mockApiInterceptor] : []),
        authInterceptor,
        errorInterceptor
      ])
    )
  ]
};
```

### Angular 14 (`app.module.ts`)

```typescript
import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { MockApiInterceptor } from './core/mock/mock-api.interceptor';
import { environment } from '../environments/environment';

providers: [
  // Mock first so it runs before auth
  ...(environment.useMockData
    ? [{ provide: HTTP_INTERCEPTORS, useClass: MockApiInterceptor, multi: true }]
    : []),
  { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true },
  { provide: HTTP_INTERCEPTORS, useClass: ErrorInterceptor, multi: true },
]
```

## Naming Conventions

| Item | Convention | Example |
|------|-----------|---------|
| Mock data constant | `MOCK_<ENTITY_PLURAL>` | `MOCK_ORDERS` |
| Mock list response | `MOCK_<ENTITY_PLURAL>_LIST` | `MOCK_ORDERS_LIST` |
| Mock auth token | `MOCK_AUTH_RESPONSE` | — |
| Mock interceptor (functional) | `mockApiInterceptor` | — |
| Mock interceptor (class) | `MockApiInterceptor` | — |
| Mock file | `mock-data.ts` | — |
| Interceptor file | `mock-api.interceptor.ts` | — |

## Key Rules

1. **Types come from model files** — never redefine shapes inline; always import from `../models/*.model.ts`.
2. **No `any`** — all mock response bodies must be typed.
3. **Prod guard** — `environment.useMockData` is always `false` in `environment.prod.ts`; the interceptor double-checks at runtime.
4. **Mock interceptor first** — it must be registered before auth/error interceptors so it can short-circuit.
5. **Fallthrough** — unmatched URLs call `next(req)` so partial backends work alongside mocks.
6. **Paginate properly** — use `page` and `size` query params from the request, not hardcoded offsets.
7. **All enum values reachable** — cycle enum values so every state is testable without editing the file.
