# /generate-mock-data

Generate a fully typed mock data layer so every Angular feature can be tested without a running backend.
Creates realistic in-memory data, an HTTP interceptor that serves it, and an environment toggle to switch
between mock and real API with a single flag change.

## Usage

```
/generate-mock-data [options]
```

### Options

| Flag | Effect |
|------|--------|
| `--count <n>` | Number of records to generate per entity (default: 10) |
| `--delay <ms>` | Simulated network delay in milliseconds (default: 300) |
| `--domain <name>` | Generate mock data only for this domain (e.g. `--domain orders`) |
| `--no-auth` | Skip mocking auth endpoints (login, register, logout) |
| `--reset` | Overwrite any existing mock files without prompting |

### Examples

```bash
# Generate mock data for all domains discovered in the project
/generate-mock-data

# 20 records per entity, 500ms simulated delay
/generate-mock-data --count 20 --delay 500

# Only mock the orders domain
/generate-mock-data --domain orders

# Reset / regenerate everything from scratch
/generate-mock-data --reset
```

## Step 3 — Generate Mock Data

Read the `skills/mock-data.md` skill for field generation rules.

Create `src/app/core/mock/mock-data.ts`:

### Rules

- Generate `--count` records (default 10) per entity.
- Every record must have a unique `id` field (use sequential strings like `"ord-001"`, `"prd-001"`, or UUIDs — match the model's id type).
- All enum/union fields cycle through every possible value across the dataset (e.g. if status has 5 values and count is 10, each value appears twice).
- Date fields use recent realistic ISO 8601 strings spread across the past 90 days.
- Number fields use plausible ranges based on field names (price/amount: 5–500; quantity: 1–20; rating: 1–5; percentage: 0–100).
- String fields use realistic domain-appropriate content (names, titles, descriptions — never "string1" or "test").
- Nested objects are fully populated (not null unless the model marks them optional).
- List responses (`*ListResponse` types) wrap items in `{ data: [...], total, page, size }`.

### Output structure

## Step 4 — Generate Mock API Interceptor

Create `src/app/core/mock/mock-api.interceptor.ts`.

### Interceptor behaviour

- Only activates when `environment.useMockData === true`.
- Matches each incoming request by **HTTP method + URL pattern**.
- Returns `of(response).pipe(delay(MOCK_DELAY))` — simulates real network latency.
- Supports pagination: reads `page` and `size` from query params and slices the mock array.
- For unknown URLs (not in the mock routes) — passes through to the real HTTP backend (`next(req)`).
- All mock responses are typed — no `any`.

### URL matching logic

Extract the last path segment(s) to determine intent:

| Pattern | Method | Returns |
|---------|--------|---------|
| `/api/<entity>` | GET | `*ListResponse` (paginated) |
| `/api/<entity>/:id` | GET | single item by id |
| `/api/<entity>` | POST | newly "created" item (echo dto + new id) |
| `/api/<entity>/:id` | PUT or PATCH | merged/updated item |
| `/api/<entity>/:id` | DELETE | `null` (204 No Content) |
| `/api/auth/login` | POST | `MOCK_AUTH_RESPONSE` |
| `/api/auth/logout` | POST | `null` |
| `/api/auth/me` | GET | `MOCK_USERS[0]` |

### Template

```typescript
// src/app/core/mock/mock-api.interceptor.ts
import { HttpInterceptorFn, HttpResponse } from '@angular/common/http';
import { inject } from '@angular/core';
import { of } from 'rxjs';
import { delay } from 'rxjs/operators';
import { environment } from '../../../environments/environment';
import {
  MOCK_ORDERS, MOCK_ORDERS_LIST,
  MOCK_PRODUCTS, MOCK_PRODUCTS_LIST,
  MOCK_AUTH_RESPONSE, MOCK_USERS
} from './mock-data';

const MOCK_DELAY = environment.mockDelay ?? 300;

export const mockApiInterceptor: HttpInterceptorFn = (req, next) => {
  if (!environment.useMockData) {
    return next(req);
  }

  const url = req.url;
  const method = req.method;

  // ── Auth ────────────────────────────────────────────────────────────────────
  if (url.includes('/auth/login') && method === 'POST') {
    return of(new HttpResponse({ status: 200, body: MOCK_AUTH_RESPONSE })).pipe(delay(MOCK_DELAY));
  }
  if (url.includes('/auth/logout') && method === 'POST') {
    return of(new HttpResponse({ status: 204, body: null })).pipe(delay(MOCK_DELAY));
  }
  if (url.includes('/auth/me') && method === 'GET') {
    return of(new HttpResponse({ status: 200, body: MOCK_USERS[0] })).pipe(delay(MOCK_DELAY));
  }

  // ── Orders ──────────────────────────────────────────────────────────────────
  if (url.match(/\/orders\/[\w-]+$/) && method === 'GET') {
    const id = url.split('/').pop()!;
    const item = MOCK_ORDERS.find(o => o.id === id) ?? MOCK_ORDERS[0];
    return of(new HttpResponse({ status: 200, body: item })).pipe(delay(MOCK_DELAY));
  }
  if (url.includes('/orders') && method === 'GET') {
    const params = req.params;
    const page = +(params.get('page') ?? 1);
    const size = +(params.get('size') ?? 20);
    const start = (page - 1) * size;
    const data = MOCK_ORDERS.slice(start, start + size);
    return of(new HttpResponse({ status: 200, body: { data, total: MOCK_ORDERS.length, page, size } })).pipe(delay(MOCK_DELAY));
  }
  if (url.includes('/orders') && method === 'POST') {
    const body = req.body as Partial<typeof MOCK_ORDERS[0]>;
    const newItem = { ...MOCK_ORDERS[0], ...body, id: `ord-${Date.now()}`, createdAt: new Date().toISOString() };
    return of(new HttpResponse({ status: 201, body: newItem })).pipe(delay(MOCK_DELAY));
  }
  if (url.match(/\/orders\/[\w-]+$/) && (method === 'PUT' || method === 'PATCH')) {
    const id = url.split('/').pop()!;
    const item = MOCK_ORDERS.find(o => o.id === id) ?? MOCK_ORDERS[0];
    const updated = { ...item, ...req.body as object, updatedAt: new Date().toISOString() };
    return of(new HttpResponse({ status: 200, body: updated })).pipe(delay(MOCK_DELAY));
  }
  if (url.match(/\/orders\/[\w-]+$/) && method === 'DELETE') {
    return of(new HttpResponse({ status: 204, body: null })).pipe(delay(MOCK_DELAY));
  }

  // ── Products (same pattern — generated per domain) ───────────────────────
  // ... (generated for each domain following the same pattern above)

  // ── Fallthrough — unknown URL, pass to real HTTP ─────────────────────────
  return next(req);
};
```

Generate a block for **every domain** following this same CRUD pattern.

## Step 5 — Update Environment Files

### `src/environments/environment.ts`

Add `useMockData: true` and `mockDelay: 300` (or the value from `--delay`):

```typescript
export const environment = {
  production: false,
  apiUrl: '/api',
  useMockData: true,      // ← set to false to hit real backend
  mockDelay: 300          // ← ms delay on mock responses
};
```

If the file already has `useMockData`, update the value. Do not duplicate the key.

### `src/environments/environment.prod.ts`

Always set `useMockData: false` in production:

```typescript
export const environment = {
  production: true,
  apiUrl: 'https://api.yourapp.com',
  useMockData: false,
  mockDelay: 0
};
```

## Step 6 — Register Mock Interceptor

### Angular 15+ (standalone / functional interceptors)

Update `src/app/app.config.ts`:

```typescript
import { provideHttpClient, withInterceptors } from '@angular/common/http';
import { authInterceptor } from './core/interceptors/auth.interceptor';
import { errorInterceptor } from './core/interceptors/error.interceptor';
import { mockApiInterceptor } from './core/mock/mock-api.interceptor';
import { environment } from '../environments/environment';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient(
      withInterceptors([
        ...(environment.useMockData ? [mockApiInterceptor] : []),
        authInterceptor,
        errorInterceptor
      ])
    )
  ]
};
```

**The mock interceptor must be first** — it short-circuits the request before auth/error interceptors run.

### Angular 14 (NgModule)

Update `src/app/app.module.ts`:

```typescript
import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { MockApiInterceptor } from './core/mock/mock-api.interceptor';
import { environment } from '../environments/environment';

@NgModule({
  providers: [
    ...(environment.useMockData
      ? [{ provide: HTTP_INTERCEPTORS, useClass: MockApiInterceptor, multi: true }]
      : []),
    { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true },
    { provide: HTTP_INTERCEPTORS, useClass: ErrorInterceptor, multi: true },
  ]
})
export class AppModule {}
```

For Angular 14, generate `MockApiInterceptor` as a class implementing `HttpInterceptor` instead of a functional interceptor.

## Notes

- The mock interceptor only activates in environments where `useMockData: true`. Production builds always use `useMockData: false`.
- Mock data types are imported from your actual model files — TypeScript will catch any mismatch at compile time.
- The interceptor short-circuits before the auth interceptor, so login works in mock mode without a real token.
- Unknown URLs fall through to the real backend — useful when only some domains are ready.
- Re-running `/generate-mock-data --reset` regenerates all files; your manual edits to `mock-data.ts` will be overwritten.
- To preserve custom mock edits across regeneration, add your entries **after** the generated block — annotate with `// custom`.
